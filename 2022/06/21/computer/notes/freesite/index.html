# 编写网站

直接上目录结构

如图所示，你需要一个 node.js
然后需要在根目录下创建一个名为 vercel.json 的配置文件

{
  "headers": [{
    "source": "/(.*)",
    "headers" : [
      {
        "key" : "Access-Control-Allow-Origin",
        "value" : "*"
      },
      {
        "key" : "Access-Control-Allow-Headers",
        "value" : "content-type"
      },
      {
        "key" : "Access-Control-Allow-Methods",
        "value" : "DELETE,PUT,POST,GET,OPTIONS"
      }
    ]
  }],
  "rewrites": [
    {
      "source": "/",
      "destination": "/api/index"
    }
  ]
}

# 函数如何编写？

函数可以简单理解为服务器暴露给外部的服务接口，api

函数全部放在 api 目录下！！！
文件名就是路径名！！！！！！！！！！！
所以聪明的你就知道了，访问 域名 /api/xxx，就会调用 rootDir/api/xxx.js !!
呐呐，一个 xxx.js 长这样，属于 node 的模块哦

值得注意的是此处的 req res 和 node 常用的那个 res 和 res 不同
这里的 req 只能获取 req.query、req.body 等参数，具体看 官方文档 吧
